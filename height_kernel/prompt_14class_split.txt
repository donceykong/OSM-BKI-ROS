## Phase 3: Tune height kernel parameters

With the 14-class split working, tune for maximum improvement. The height kernel now has two separate levers:
- Terrain (class 9, mu=0, tau=0.5): suppress false terrain at height → helps building, pole, traffic-sign
- Vegetation (class 13, mu=8, tau=5): suppress false vegetation at ground → helps road, sidewalk, terrain

Start with the cross-domain config (kitti360_with_height.yaml) since it has the most room for improvement. Current params already work well — the split itself should give a big boost.

For in-domain, the key insight: the dominant error was "everything→vegetation" which was unfixable because vegetation spanned all heights. NOW vegetation means only trees (h≈8m). So false vegetation predictions at ground level (which were really terrain confusion) can be suppressed.

Tune these parameters in the *_with_height configs:
- height_kernel_lambda: start at 0.9 for cross-domain, 0.8 for in-domain
- dead_zone: 1.5 for cross-domain (already set), 0.5-0.75 for in-domain
- terrain tau: 0.3-0.5 (tight, ground level)
- vegetation tau: 4.0-6.0 (moderate, trees vary in height)
- Use redistribute: true for in-domain (already set)
