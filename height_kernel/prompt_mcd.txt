## Height Kernel Formula
For each voxel at height h above ground (from DEM):
  ybars[k] *= (1-lambda) + lambda * exp(-(h - mu_k)^2 / (2 * tau_k^2))

Where h = voxel_z - dem_grid_value. Applied per-class in bkioctomap.cpp.
Also has DEM/DSM occupancy prior: ybars[0] += dem_occupancy_strength for voxels above DSM or below DEM.

## Parameters to Tune (in the *_with_height.yaml configs)
- height_kernel_lambda: strength (0=off, 1=full authority)
- height_kernel_mu: per-class expected height [13 values for common taxonomy]
- height_kernel_tau: per-class tolerance [13 values] — larger = more permissive
- dem_occupancy_strength: free-space evidence (0=disabled)
- dem_occupancy_margin: tolerance in meters

## Class Order (common taxonomy, 13 classes)
0:unlabeled, 1:road, 2:sidewalk, 3:parking, 4:other-ground, 5:building, 6:fence, 7:pole, 8:traffic-sign, 9:vegetation, 10:two-wheeler, 11:vehicle, 12:other-object

### What was tried (all failed on MCD):
1. lambda=0.9, tight taus (original KITTI-360 tuning) → mIoU -6.23% (exp3), -12.31% (exp4). Catastrophic.
2. lambda=0.3, wider taus [5,5,5,5,8,5,8,6,15,3,3,5] → mIoU -4.21% (exp3), -9.00% (exp4). Still bad.
3. lambda=0.15, wide taus, occupancy=0.2 → mIoU -3.81% (exp3). DEM occupancy was the main culprit.
4. lambda=0.15, wide taus, occupancy=0.0 → mIoU -0.32% (exp3), -0.39% (exp4). Nearly neutral! Overall Acc +1.71% for exp3. Vehicle was the big loser at -7.86%.
5. lambda=0.25, vehicle/2wheeler tau=100 (disabled), building tau=10, veg tau=15 → never ran (was queued)
6. lambda=0.7, building tau=4, veg tau=6 only, rest tau=100 → mIoU -4.05% (exp3). Tau too tight for noisy DEM.
7. lambda=0.5, building tau=8, veg tau=8, rest tau=100 → mIoU -2.12% (exp3). Still too tight.
8. lambda=0.5, building tau=15, veg tau=12, rest tau=100 → currently configured, not yet run

### Key Insight
The ONLY configuration that came close to helping was #4 (lambda=0.15, all taus wide, no occupancy). It gave +1.71% overall accuracy on exp3 but slightly negative mIoU. The kernel can only SUPPRESS (multiply by factor ≤ 1), never boost. So it helps by suppressing wrong predictions at wrong heights, but hurts if it also suppresses correct predictions.

### Possible Approaches to Try
1. **Ultra-conservative**: lambda=0.1-0.2, only building+vegetation with tau matching their height std (tau > 3*std), no occupancy
2. **Height-band approach**: Only suppress predictions that are VERY far from expected height (e.g., building at h<1 or vegetation at h<1)
3. **Asymmetric approach**: Maybe modify the C++ code to only apply the kernel when the suppression would actually help (e.g., only suppress low-confidence predictions)
4. **Fix the DEM**: The 23% NaN coverage might be improvable by interpolating from valid cells
5. **Reconsider the occupancy prior**: With margin=3.0+, maybe it could work — the issue was margin=1.5 was too tight
