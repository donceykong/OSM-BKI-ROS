## Height Kernel Formula (in src/mapping/bkioctomap.cpp, line ~366)
```
h = voxel_z - dem_grid_value  (height above ground from DEM)

For each class k:
  abs_diff = |h - mu_k|
  if abs_diff <= dead_zone: phi_k = 1.0 (no effect)
  else: phi_k = exp(-((abs_diff - dead_zone)^2) / (2 * tau_k^2))

  ybars[k] *= (1 - lambda) + lambda * phi_k
```

Applied after OSM prior, before Dirichlet update. Also has:
- `gate`: if phi of argmax class >= gate, skip kernel entirely (only fire on incompatible predictions)
- `redistribute`: alternative mode that reweights rather than suppresses
- `dem_occupancy_strength/margin`: free-space prior for voxels above DSM or below DEM

## Parameters to Tune (in *_with_height.yaml configs)
- height_kernel_lambda: strength (0=off, 1=full)
- height_kernel_dead_zone: points within this distance of mu_k are untouched
- height_kernel_gate: if argmax phi >= gate, skip kernel (0=disabled)
- height_kernel_redistribute: true=reweight mode, false=suppress mode
- height_kernel_mu: per-class expected height [13 values]
- height_kernel_tau: per-class tolerance [13 values], larger=more permissive, 100000=disabled
- dem_occupancy_strength: free-space evidence (0=disabled)
- dem_occupancy_margin: tolerance in meters

Root cause: When in-domain predictions are already good (27-31% mIoU baseline), the height kernel overrides correct answers. The kernel can only SUPPRESS (multiply ≤ 1), never boost.

## What Worked for MCD (From Agent 1 — DO NOT CHANGE MCD CONFIGS)
Agent 1 found that:
- dead_zone parameter is critical — points within dz meters of expected height should NOT be touched
- Only activating specific ground classes (sidewalk tau=2.5-4.0) while disabling everything else (tau=100000) works
- gate parameter helps for cross-domain but not in-domain
- DEM occupancy must be disabled (strength=0) for noisy DEM — but KITTI-360 DEM is high quality so it might work here

## Strategy Suggestions
1. Start with the cross-domain configs — they already work at full scale. Add dead_zone to make them even better, then validate at 15 scans.
2. For in-domain, try the dead_zone + selective class approach:
   - dead_zone=2.0 (KITTI-360 DEM is accurate, so 2m is generous)
   - Only activate classes that have clear height separation from their confusion targets
   - Building (mu=7, tau=4): suppress false building at ground level
   - Sidewalk (mu=0.4, tau=2.5): suppress false sidewalk at height
   - Everything else: tau=100000 (disabled)

3. Consider the gate parameter: gate=0.7 means kernel only fires when the top prediction is height-incompatible
